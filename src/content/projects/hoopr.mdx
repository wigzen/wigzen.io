## The Challenge

Hoopr.ai is a music licensing platform connecting creators with licensable music. The core challenge was building a system that could handle real-time music discovery across a massive catalog while maintaining fast page loads and an intuitive search experience.

The platform needed to serve 40,000+ creators with instantaneous search, seamless audio previewing, and a licensing workflow that didn't break flow. Every millisecond of latency in search or page load directly impacted creator conversion.

## Architecture

The system was designed as a layered architecture:

**Frontend Layer** — React with Next.js for server-rendered pages (SEO-critical catalog pages) and client-side interactive elements (search, audio player, licensing flow). Component architecture allowed reuse across catalog, search, and artist pages.

**API Layer** — Node.js backend with RESTful endpoints for search, catalog management, and licensing. Redis caching layer reduced database load and brought search responses under 200ms at p95.

**Data Layer** — PostgreSQL as the primary datastore for catalog metadata, licensing records, and user data. Structured for efficient full-text search with optimised indexes.

**Delivery Layer** — CDN-optimised audio delivery with lazy loading. Audio files served through edge nodes with progressive streaming for instant previews.

## Systems Lens

This project demanded thinking across the full stack — from database query optimisation to CDN cache invalidation to frontend rendering strategies. The key insight was that music discovery is latency-sensitive: users expect instant results. This drove every architectural decision toward aggressive caching, edge delivery, and progressive loading.

## Outcome

The platform shipped with measurable improvements: 35% increase in user engagement, 25% faster page loads, 30% improvement in platform scalability, and 20% boost in user retention. The architecture scaled to serve the growing creator base without degradation.
